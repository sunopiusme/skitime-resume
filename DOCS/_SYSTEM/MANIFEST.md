# Manifest

Ключевые слова `MUST`, `SHOULD`, `MAY` интерпретируются по BCP 14 / RFC 8174, когда записаны в верхнем регистре.

## Назначение

Этот manifest определяет документацию `skitime-resume` как слой решений для персонального сайта, который должен выглядеть как продуктовый репозиторий, а не как набор красивых секций.

## Принципы

- Каждый тип решения MUST иметь один явный source of truth.
- Публичный сайт MUST демонстрировать инженерную серьёзность даже при чтении исходников.
- Документация MUST предшествовать спорным архитектурным и визуальным решениям, а не объяснять их постфактум.
- Кодовая база MUST избегать декоративных комментариев и генераторного мусора.
- Контент MAY быть компактным, но не расплывчатым и не placeholder-driven.
